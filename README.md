# AWS Multi-Region Disaster Recovery Architecture Using Route 53 Failover Routing

## Project Overview

This project demonstrates the implementation of a Multi-Region Disaster Recovery (DR) architecture on AWS. The application is deployed in two AWS Regions:

* Primary Region: Mumbai (ap-south-1)
* Secondary Region: North Virginia (us-east-1)

AWS Route 53 Failover Routing Policy and Health Checks are used to automatically redirect traffic to the secondary region whenever the primary region becomes unavailable.

The objective of this project is to ensure High Availability, Business Continuity, and Disaster Recovery using AWS native services.

## Project Workflow

### Step 1: Launch Primary EC2 Instance in Mumbai

Region:

ap-south-1

Install NGINX:

```bash
sudo apt update
sudo apt install nginx -y
sudo systemctl start nginx
sudo systemctl enable nginx
```

Verify:

```bash
systemctl status nginx
```

Access:

```text
http://<Mumbai-EC2-Public-IP>
```

---

### Step 2: Host Sample Website

Update NGINX default page:

```bash
sudo nano /var/www/html/index.html
```

Content:

```html
<h1>Primary Site - Mumbai Region</h1>
```

---

### Step 3: Create AMI

Navigate:

EC2 → Instances → Select Instance → Actions → Image and Templates → Create Image

Name:

```text
Mumbai-WebServer-AMI
```

Wait until AMI status becomes:

```text
Available
```

---

### Step 4: Copy AMI to North Virginia

Navigate:

EC2 → AMIs → Select AMI → Actions → Copy AMI

Destination Region:

```text
us-east-1
```

Wait until copied AMI becomes available.

---

### Step 5: Launch DR EC2 Instance

Launch new EC2 using copied AMI.

Region:

```text
us-east-1
```

Update webpage:

```html
<h1>Disaster Recovery Site - North Virginia</h1>
```

Restart NGINX:

```bash
sudo systemctl restart nginx
```

---

### Step 6: Create Application Load Balancer in Mumbai

Navigate:

EC2 → Load Balancer → Create Load Balancer

Type:

```text
Application Load Balancer
```

Configuration:

* Internet Facing
* HTTP Listener (Port 80)
* Public Subnets
* Security Group Allowing Port 80

Create Target Group:

```text
tg-mumbai
```

Register Mumbai EC2 Instance.

Verify:

```text
Target Health = Healthy
```

---

### Step 7: Create Application Load Balancer in North Virginia

Repeat same process.

Create:

```text
tg-virginia
```

Register Virginia EC2.

Verify:

```text
Target Health = Healthy
```

---

### Step 8: Verify ALB Access

Open:

```text
http://<Mumbai-ALB-DNS>
```

Expected:

```text
Primary Site - Mumbai Region
```

Open:

```text
http://<Virginia-ALB-DNS>
```

Expected:

```text
Disaster Recovery Site - North Virginia
```

---

### Step 9: Create Route 53 Hosted Zone

Navigate:

Route 53 → Hosted Zones

Create Hosted Zone:

```text
evertrendmart.in
```

Type:

```text
Public Hosted Zone
```

---

### Step 10: Configure Domain Nameservers

Update Nameservers at Domain Registrar.

Use Route 53 Nameservers generated by AWS.

Example:

```text
ns-xxx.awsdns.com
ns-xxx.awsdns.net
ns-xxx.awsdns.org
ns-xxx.awsdns.co.uk
```

Wait for DNS propagation.

---

### Step 11: Create Health Check

Navigate:

Route 53 → Health Checks

Configuration:

Protocol:

```text
HTTP
```

Endpoint:

```text
Mumbai ALB DNS
```

Port:

```text
80
```

Path:

```text
/
```

Verify status becomes:

```text
Healthy
```

---

### Step 12: Create Primary Failover Record

Route 53 → Create Record

Configuration:

* Type: A Record
* Alias: Yes
* Routing Policy: Failover
* Record Type: Primary
* Target: Mumbai ALB
* Attach Health Check

Save Record.

---

### Step 13: Create Secondary Failover Record

Configuration:

* Type: A Record
* Alias: Yes
* Routing Policy: Failover
* Record Type: Secondary
* Target: Virginia ALB

Save Record.

## Failover Testing

### Normal Condition

Traffic Flow:

Users → Route53 → Mumbai ALB → Mumbai EC2

Website Displays:

```text
Primary Site - Mumbai Region
```

---

### Simulate Region Failure

Stop NGINX:

```bash
sudo systemctl stop nginx
```

or Stop EC2 Instance.

Health Check Status Changes:

```text
Healthy → Unhealthy
```

Route53 Automatically Redirects Traffic:

Users → Route53 → Virginia ALB → Virginia EC2

Website Displays:

```text
Disaster Recovery Site - North Virginia
```

---

### Recovery Test

Start NGINX:

```bash
sudo systemctl start nginx
```

Health Check Returns:

```text
Healthy
```

Traffic Automatically Returns to Mumbai Region.

## Troubleshooting Performed During Project

### Issue 1: ALB DNS Not Loading

Error:

```text
ERR_CONNECTION_TIMED_OUT
```

Resolution:

* Verified ALB was Internet Facing
* Verified Security Group allowed Port 80
* Verified ALB was deployed in Public Subnets
* Verified Route Table had Internet Gateway Route
* Verified Target Group Health Status

---

### Issue 2: Target Group Unhealthy

Resolution:

Verified:

```bash
sudo systemctl status nginx
```

Verified application accessibility:

```bash
curl localhost
```

Corrected Health Check Path:

```text
/
```

---

### Issue 3: Route 53 Domain Not Resolving

Resolution:

Verified Route 53 Nameservers were correctly configured at Domain Registrar.

Checked DNS propagation.

Verified using:

```bash
nslookup evertrendmart.in
```

---

### Issue 4: Incorrect Telnet Testing

Incorrect:

```bash
telnet http://alb-dns-name 80
```

Correct:

```bash
telnet alb-dns-name 80
```
